# v0.2.0 — Decision #21 hooks shipped

First public release of un-punt.

## What it does

Captures every deferral your AI coding agent makes — `// TODO`, `// FIXME`, `as any`, `.skip`, swallowed exceptions, "I'll handle this later" — as a typed markdown item in `.un-punt/items/<id>.md`. Proposes a verified cleanup pass at natural stopping points with disposition-gated commits (per-item / squashed / side-branch / uncommitted). Refuses what it can't safely fix (auth, crypto, payments, migrations, generated code, anything human-touched in the last 24h).

Plain markdown. No DB. No daemon. No MCP server. The skill is the IP.

## How to install

### Claude Code via the marketplace

```text
/plugin marketplace add bcanfield/un-punt
/plugin install un-punt@un-punt
```

### Or via npm

```bash
npx un-punt install claude-code
```

Either path merges the permission + hook blocks into `~/.claude/settings.json`, copies the skill to `~/.claude/skills/un-punt/`, and drops a contract template into `<cwd>/.un-punt/contract.md`. Re-running install is idempotent. `un-punt uninstall` reverses every settings mutation precisely.

## What's in the box

- **Skill body** — `~/.claude/skills/un-punt/SKILL.md`. Loaded at session start by a SessionStart hook (see Decision #21).
- **Three hooks** — `SessionStart`, `PostToolUse`, `UserPromptSubmit` — emit context reminders to the agent on deterministic events. The hooks do **no content classification** — that stayed with the agent.
- **Contract template** — `<cwd>/.un-punt/contract.md` per repo. Carries the trust contract (refuse > flag > fix) and configurable thresholds.
- **Refusal lists** — categorical (auth/crypto/payments/migrations/lockfiles/generated-code) and hostile-input.
- **CLI** — `un-punt install <platform>` / `un-punt status` / `un-punt uninstall`. Tracked install manifest at `~/.claude/un-punt-install.json` so uninstall is precisely reversible.

## Why v0.2 and not v0.1

We shipped v0.1 (skill-only, no hooks) per a careful design process and then dogfooded it on a real product build for two weeks. The skill was empirically dormant — it never auto-loaded on coding-topic conversations and never silently captured during normal Edit/Write tool calls. Cold-start (`/un-punt`) recovered everything once invoked, but the "always-on" promise was empirically false.

v0.2 ships **Decision #21**: three hooks fire at deterministic events; the agent applies the skill body's rules. The four failed v0.1 probes pass under v0.2.

## Cross-platform

Cursor, Codex, Gemini CLI all shipped stable hook systems with Claude-Code-compatible contracts during 2025–2026. The same skill body and same hook scripts port to those platforms — only the registration files differ. Cursor + Codex adapters land in v0.2.x patches.

## Known limitations

- **Single-developer-per-repo** at MVP (Decision #12).
- **Bypass mode (`--dangerously-skip-permissions`)** — hooks fire normally and skill operates as expected. Reduced: OS-level `permissions.deny` enforcement on categorical refusals; load-bearing safety in that mode is the skill body's interpretive refusals + disposition prompt. v0.3 plans a `PreToolUse` refusal-floor hook for stronger bypass-mode safety (see `docs/v0.3-roadmap.md` V03-10).
- **Description-match auto-loading is fragile** without the hooks. Decision #21 made hooks the default; if you bypass the install CLI and skip hook registration, expect the same dormancy v0.1 had.
